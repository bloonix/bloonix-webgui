%%site.help.doc.host-parameter%%

Formularfeld: %%schema.company.attr.company%%

Benutzer mit der Rolle admin können hier den Host einer Unternehmsgruppe zuordnen. Sollten Sie nur die Rolle operator haben, bleibt Ihnen wahrscheinlich nur die Angabe der Unternehmensgruppe, der Sie selbst angehören.

Formularfelder: %%schema.host.attr.hostname%%, %%schema.host.attr.ipaddr%%

Diese Parameter sollten selbsterklärend sein. Zu beachten ist allerdings, dass der Hostname eindeutig sein muss.

Formularfelder: %%schema.host.attr.description%%, %%schema.host.attr.comment%%

Diese zwei Felder sind selbsterklärend. Setzen Sie eine Beschreibung des Hosts wie zum Beispiel "Webserver" oder "Mailserver". Als Kommentar eignen sich besonders kurzlebige Informationen wie zum Beispiel "Es finden Wartungsarbeiten statt" oder "in Bearbeitung von Admin Tom". Sobald ein Feld gesetzt ist, erscheint es auch in den Statusmails.

Formularfeld: %%schema.host.attr.password%%

Dieses Passwort wird vom Bloonix-Agenten verwendet, damit dieser sich mit der ID und dem Passwort des Hosts beim Bloonix-Server authentifizieren kann und um Daten zu Services für den Host abliefern zu können.

Formularfeld: %%schema.host.attr.sysinfo%%

Mit dem Parameter %%schema.host.attr.sysinfo%% haben Sie die Möglichkeit, einen externen Link einzutragen. Am besten eignet sich die Angabe eines Links zu einer firmeninternen Hardwareerfassung.

Beispiel für den Parameter %%schema.host.attr.sysinfo%%: 

12345=https://mysite.test/?id=12345

Der Parameter hat den Aufbau: 

Alias=Link

Der Alias wird dabei in den Beschreibungen zum Host als Link angezeigt.

Formularfelder: %%schema.host.attr.sysgroup%%, %%schema.host.attr.host_class%%, %%schema.host.attr.system_class%%, %%schema.host.attr.location_class%%, %%schema.host.attr.hw_manufacturer%%, %%schema.host.attr.hw_product%%, %%schema.host.attr.os_manufacturer%%, %%schema.host.attr.os_product%%, %%schema.host.attr.virt_manufacturer%%, %%schema.host.attr.virt_product%%, %%schema.host.attr.location%%

Mit diesen Feldern können Sie genauere Angaben zu einem Host machen. Zum Beispiel ob es sich bei dem Host um einen Server, einen Drucker oder einen Router handelt. Ebenfalls können Sie Informationen zum Betriebssystemhersteller oder der Version eintragen.

Bitte beachten Sie, dass diese Felder über eine Autovervollständigung verfügen. Das heisst, dass wenn Sie die Felder einmal für einen Host ausgefüllt haben, e rhalten Sie beim nächsten Host die bereits existierenden Werte als Vorschlag. Falls die Liste der Einträge zu lang ist, dann tippen Sie ein paar Buchstaben ein um die Anzeige zu verfeinern.

Zu den Parametern %%schema.host.attr.host_class%%, %%schema.host.attr.system_class%% und %%schema.host.attr.location_class%% gibt es weitere Informationen im folgenden Dokument: %%site.help.doc.host-classes%%

Formularfeld: %%schema.host.attr.coordinates%%

Bitte wählen Sie hier den genauen Standort Ihres Servers aus. Dieser Wert wird verwendet, um Ihren Host auf einer Weltkarte anzuzeigen.

Formularfeld: %%schema.host.attr.active%%

Mit diesem Parameter können Sie einen Host aktivieren oder deaktivieren. Mit der Deaktivierung eines Hosts werden alle Services eines Hosts nicht mehr überwacht.

Formularfeld: %%schema.host.attr.allow_from%%

Dieser Parameter sorgt in Kombination mit dem Passwort für optimale Sicherheit. Mit dem Parameter kann festgelegt werden, von welcher IP-Adresse ein Agent Daten für den Host abliefern darf. Die IP-Adresse 127.0.0.1 ist standardmäßig immer freigeschaltet.

Falls der Bloonix-Agent von mehr als einer IP-Adresse aus Daten abliefern kann, dann können Sie hier eine kommaseparierte Liste von IP-Adressen eintragen. Der Eintrag "all" sorgt dafür, dass alle IP-Adressen freigeschaltet sind.

Formularfeld: %%schema.host.attr.interval%%

Der Intervall hat einen entscheidenen Verwendungszweck bei der Erstellung der RRDs (Round Robin Databases). Der Wert sollte analog zum Parameter "interval" eingestellt werden, der beim zuständigen Bloonix-Agenten eingerichtet ist. Ist also der Bloonix-Agent, der die Checks für diesen Hosts ausführt, zum Beispiel auf einen Intervall von 60 Sekunden eingestellt, dann sollte dieser Wert ebenfalls auf 60 Sekunden gesetzt werden.

Formularfeld: %%schema.host.attr.timeout%%

Mit diesem Wert wird bestimmt, nach wievielen Sekunden der Status eines Service als veraltet markiert wird und sollte um ein vielfaches von "Intervall" gesetzt werden. Das ist zum Beispiel dann wichtig, wenn ein Bloonix-Agent aus irgendeinem Grund den Status eines Service nicht mehr aktualisiert, vielleicht weil er ausgefallen oder zu stark überlastet ist.

Zu beachten ist, dass dieser Wert für alle Services gilt. Sollte einmal ein Service tatsächlich auf diesem Timeout laufen, dann wird der Status des Service auf folgende Meldung gesetzt: 

Service check timeout after 180s (host/agent dead?)

Formularfeld: %%schema.host.attr.max_services%%

Falls Sie den Wunsch haben, die maximal konfigurierbaren Services für einen Host festzulegen, dann ist das über diesen Parameter möglich. Der Wert 0 bedeutet, dass es keine Einschränkung gibt.

Formularfeld: %%schema.host.attr.max_sms%%

Mit dieser Option können Sie kontrollieren, wieviele SMS pro Monat höchstens für einen Host mit all seinen Services versendet werden dürfen. Ist das Maximum erreicht, dann werden keine Warnmeldungen mehr per SMS versendet. Natürlich erhalten Sie in jeder SMS und Mail eine zusächtliche Warnung, sollten nur noch 10% oder weniger als 5 SMS verfügbar sein.

In Statusmails sieht die Warnung wie folgt aus: 

=== WARNING ===
460/500 SMS used
Please increase the maximal allowed SMS per month!

Per SMS erhalten Sie selbstverständlich eine abgekürzte Warnmeldung: 

Hostname ip-addresse [461/500 SMS USED] status

Von welchem Standort aus soll der Check ausgeführt werden?

Mit den Standort-Checks haben Sie die Möglichkeit festzulegen, welcher Bloonix-Agent an welchem Standort Ihren Service prüfen soll.

Bloonix selbst bietet zum Beispiel diverse Standorte in Deutschland an, um Ihre Webservices per HTTP zu prüfen. Sie könnten diese Services aber genauso von eigenen Servern testen lassen, wenn dort ein Bloonix-Agent läuft und für die entsprechenden Services konfiguriert ist.

Es gibt auch Service-Checks, für die Sie definitiv einen Bloonix-Agenten auf Ihrem Server installieren müssen. Da wären zum Beispiel Prüfungen, die weder über SNMP abgefragt werden können oder die über keine eigene Netzwerkverkverbindung verfügen. Es gibt aber auch Services, dessen Ports Sie eventuell nicht für das Internet erreichbar machen möchten. Für diese Service-Checks, beispielsweise zur Überwachung der CPU-Last, der Load Average, des Hauptspeichers, der Festplattenkapazität, der MySQL oder PostgreSQL Datenbank etc., ist die Installation des Bloonix-Agenten auf Ihrem Server notwendig.

Bei der Einrichtung von Services in der Weboberfläche gibt es drei mögliche Optionen zur Auswahl des Standorts: localhost, intranet und remote.

localhost

Mit der Option localhost wird der Check lokal auf Ihrem Server ausgeführt. Hierzu ist es notwendig, dass der Bloonix-Agent auf Ihrem Server installiert ist. Diese Option ist besonders sinnvoll, wenn Sie die Systemvitals, wie zum Beispiel die CPU, den Hauptspeicher oder auch die Festplatten überwachen möchten.

intranet

Mit der Option intranet ist ihr lokales Netzwerk gemeint. Hierfür ist es notwendig, dass Sie den Bloonix-Agenten in Ihrem lokalen Netzwerk auf einem zentralen Server installieren. Von diesem Server aus werden die Checks ausgeführt. Diese Option ist sinnvoll, wenn Ihre Server Services bereitstellen, welche nicht über eine Internetanbindung erreichbar sind, aber dennoch von einem anderen Server aus überprüft werden sollen. Das können zum Beispiel Router, Switches etc. sein.

remote

Mit der Option remote wird der Check von einem externen Bloonix-Server ausgeführt. Dies ist besonders für Internetservices sinnvoll. Zum Beispiel können Sie über einen externen Check die Funktionalität Ihres Webservers bzw. Ihrer Webseiten überprüfen.

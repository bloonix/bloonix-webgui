%%site.help.doc.host-templates%%

Was sind Host-Templates?

Mit den Host-Templates haben Sie die Möglichkeit, Gruppen mit Service-Checks zu definieren, die an Hosts vererbt werden. So ein Host-Template, welches zunächst nur aus einem Namen und einer Beschreibung besteht, können Sie sich wie eine Gruppe vorstellen. Dieser Gruppe können Sie Service-Checks hinzufügen, so, wie Sie auch Service-Checks zu einem Host hinzufügen würden. Nachdem ein Host-Template mit Service-Checks erstellt wurde, können Hosts dieser Gruppe zugeordnet werden. Der Host erbt dann alle Service-Checks, die in diesem Host-Template konfiguriert sind.

Wer darf Host-Templates verwalten?

Nur Benutzer mit der Rolle operator dürfen Host-Templates und Service-Checks eines Host-Templates verwalten.

Einrichtung von Host-Templates

Host-Templates lassen sich im Bereich MONITORING ⇒ Templates verwalten.

Dort können Sie neue Host-Templates anlegen und bestehende Host-Templates verwalten. Klicken Sie einfach auf den Namen eines Host-Template. Es wird sich ein Menü öffnen, über welches Sie zu den Seiten gelangen, um die Service-Checks eines Host-Template zu verwalten oder auch Hosts hinzuzufügen oder zu entfernen.

Service-Checks in Host-Templates verwalten

Das Verwalten von Service-Checks eines Host-Templates funktioniert analog zur Verwaltung von Service-Checks eines Hosts. Sie können Service-Checks hinzufügen, entfernen und die Parameter der Service-Checks anpassen.

Bitte beachten Sie: Änderungen eines Host-Templates wirken sich auf alle Hosts aus, die einem Host-Template zugeordnet sind. Das heißt, wenn Sie einen Service-Check löschen, so löschen Sie den Service-Check ebenfalls von allen Hosts, die diesem Host-Template zugeordnet sind. Das gleiche gilt für das Hinzufügen neuer Service-Checks oder das Ändern von Parametern. Wenn Sie zum Beispiel einen Schwellwert eines Service-Checks anpassen, so wird die Änderung umgehend an alle Service-Checks aller Hosts vererbt, die diesem Host-Template zugeordnet sind.

Hosts in Host-Templates verwalten

Haben Sie nun den Wunsch, einen Host mehreren Host-Templates zuzuordnen, so können Sie das an verschiedenen Stellen machen. Zum Einen können Sie direkt beim Anlegen eines neuen Hosts beliebig viele Host-Templates auswählen. Der Host erbt daraufhin umgehend alle Service-Checks der Host-Templates, die dem Host zugeordnet wurden.

Wenn Sie ein Host-Template von einem Host entfernen, so werden auch die Service-Checks mit allen Daten umgehend entfernt. Seien Sie daher vorsichtig bei der Konfiguration der Gruppen.

Host-Template Variablen

Es ist möglich, Variablen für Hosts und Host-Templates einzurichten. Diese Variablen können zur Definition von Schwellwerten eingesetzt werden. Auf diese Weise bekommen Sie ein Stück Dynamik trotz der starren Host-Templates zurück. Weitere Informationen hierzu finden Sie im Dokument: %%site.help.doc.host-variables%%

Änderung von Parametern eines Service-Checks

Die Parameter von Service-Checks, die aus einem Host-Template vererbt wurden, können nicht mehr für einzelne Hosts angepasst werden. Wenn Sie die Parameter ändern möchten, dann geht das nur über die Service-Checks, die in den entsprechenden Host-Templates konfiguriert sind.

Bitte beachten Sie, dass es möglich ist, Variablen für Hosts und Host-Templates einzurichten. Diese Variablen können zur Definition von Schwellwerten eingesetzt werden. Auf diese Weise bekommen Sie ein Stück Dynamik trotz der starren Host-Templates zurück. Weitere Informationen hierzu finden Sie im Dokument: %%site.help.doc.host-variables%%
